## 1. Compliance Rationale

In automated banking workflows, Generative AI agents must operate inside a transparent, auditable environment. Regulators (such as the Monetary Authority of Singapore - MAS, and OJK in Indonesia) require that automated modifications to financial models leave a **verifiable audit trail** proving:
1.  **Data Integrity**: Clear proof that no formula cells or static risk parameters were altered or corrupted.
2.  **Macro Preservation**: Positive validation that VBA macro binaries (`vbaProject.bin`) remain intact and unaltered.
3.  **Traceability**: A 1-to-1 linkage connecting the input source file (CSV), the AI's execution prompt, and the resulting SharePoint/Google Workspace version update.

## 3. Core Compliance Fields Explained

*   **SHA-256 Checksum**: Captures the exact cryptographic hash of the incoming CSV data. This ensures that any data processing inconsistencies can be verified against the original input source.
*   **VBA Macro Binary Hash**: Captures the hash of `xl/vbaProject.bin`. Since Excel Online does not modify macro binaries when writing cell updates, the pre-write and post-write hashes must match. If they differ, it indicates file structure changes, triggering an immediate alert.
*   **Formula Count Check**: Compares the total formula count before and after the update. A discrepancy indicates that the LLM attempted to write a raw value over an active calculation.
*   **SharePoint Version History ID**: Connects the execution log to Microsoft’s native cloud revision log, enabling the human checker to undo or restore the workbook state with a single click.
